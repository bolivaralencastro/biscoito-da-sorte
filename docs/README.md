# Diretório Docs

Este diretório contém documentação adicional sobre a arquitetura, decisões e funcionamento da aplicação "Biscoito da Sorte".

## Estrutura

```
docs/
├── ADR.md           # Decisões de Arquitetura
└── DEP_GRAPH.md     # Gráfico de Dependências
```

## Tipos de Documentação

### ADR (Arquiteture Decision Records)
- Documentos que registram decisões arquiteturais importantes
- Justificativas por trás das decisões técnicas
- Alternativas consideradas e por que foram rejeitadas

### DEP_GRAPH (Dependency Graph)
- Representação visual ou textual das dependências entre módulos
- Relacionamentos entre diferentes partes da aplicação
- Fluxos de dados e comunicação entre componentes

## Diretrizes de Documentação

### Adicionando Novos Documentos
1. Siga o formato existente (Markdown)
2. Use títulos e subtítulos claros
3. Inclua exemplos quando necessário
4. Referencie outros documentos quando apropriado

### Manutenção
- Sempre que adicionar ou modificar documentação:
  - Atualize esta documentação se necessário
  - Verifique se a informação está atualizada
  - Mantenha consistência com o restante da documentação

## Convenções de Documentação
- Use Markdown para formatação
- Inclua exemplos de código quando relevante
- Utilize diagramas ASCII ou referências a diagramas visuais
- Mantenha linguagem clara e objetiva

## Notas Importantes
- Esta documentação é crucial para o entendimento da arquitetura
- Documentos aqui devem ser mantidos atualizados com mudanças de arquitetura
- Novos membros da equipe devem consultar esta documentação

## Instruções de Manutenção
- Documente decisões arquiteturais importantes
- Atualize gráficos de dependências quando a arquitetura mudar
- Revise a documentação periodicamente para garantir atualização
- Adicione referências cruzadas entre documentos quando apropriado
